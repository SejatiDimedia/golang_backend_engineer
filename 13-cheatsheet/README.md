# Cheatsheets

Quick-reference sheets — dense, scannable, no prose. Distinct from `02-notes/`, which explains concepts; these are for fast lookup once a concept is already understood.

## Planned structure

| File | Covers |
|---|---|
| `01-syntax-cheatsheet.md` | Core Go syntax quick reference |
| `02-error-handling-cheatsheet.md` | Error wrapping/checking patterns |
| `03-concurrency-cheatsheet.md` | Goroutine/channel/context patterns |
| `04-sql-patterns-cheatsheet.md` | Common query/transaction patterns used across projects |
| `05-docker-cheatsheet.md` | Common Docker/Compose commands used in this repo |
| `06-testing-cheatsheet.md` | Table-driven tests, mocking patterns, assertions |

Populate as each topic is encountered — a cheatsheet written before the concept is used in a real project tends to be generic and low-value. Best filled in retroactively from what actually got looked up repeatedly during a project.
