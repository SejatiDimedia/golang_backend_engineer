# Lessons Learned: [Project Name]

**Written:** [date, ideally within a day or two of project completion, while it's fresh]

---

## 1. What I got wrong in the initial design, and when I noticed

Be specific. "I initially modeled X as Y, which broke down when Z happened, and I noticed during [phase]." Vague entries here ("I learned a lot about error handling") are a sign to dig one level deeper.

## 2. What I'd change if I rebuilt this today

Concrete, not aspirational. If the honest answer is "not much, I'd build it the same way," that's a valid and useful data point too — it means the design held up.

## 3. The concept that most needs reinforcement

Per `00-learning-plan.md` §9 — name the single concept from this project that felt shakiest, and where in `02-notes/` or `03-exercises/` it should get follow-up drilling before it reappears in a later project.

## 4. Which earlier project should be revisited

If this project's learning suggests an earlier project's design is now visibly outdated, name it and open the loop — either a quick note here with a planned revisit date, or directly open the revisit and log it in that project's `CHANGELOG.md`.

## 5. Estimate vs. reality

Compare against this project's `ROADMAP.md` §4 ("known risks/unknowns at planning time") and the timeline estimate in the repo-level `01-roadmap.md`. Where was the estimate wrong, and in which direction?

## 6. What surprised me

Not everything fits neatly into the four review questions above — this is open space for anything unexpected, including things that went better or were easier than expected.
