# Resources

Curated external references — books, articles, official docs, talks. Curated means selective: this is not a bookmark dump. Something earns a place here once it's actually been used and found valuable, with a one-line note on what it was useful for.

## Structure

| File | Covers |
|---|---|
| `01-go-language.md` | Core language references |
| `02-backend-architecture.md` | Clean Architecture, DDD-adjacent reading relevant to this repo's style |
| `03-databases.md` | PostgreSQL, transactions, indexing |
| `04-concurrency.md` | Goroutines, channels, concurrency patterns |
| `05-security.md` | Auth, OAuth, RBAC — most relevant ahead of Project 7 |
| `06-deployment-devops.md` | Docker, deployment practices |

## Convention

```markdown
### [Resource title](link)
**Used for:** [project/topic]
**Why it was worth it:** one line
```

Avoid adding a resource here just because it exists — add it once it's actually informed a decision somewhere in this repo, and ideally link to where.
